# 無料ツールで品質を最大化する改善計画

## 現状の問題（ユーザーフィードバック）

1. 音声がとぎれとぎれ（シーン境界でギャップ・クリック音）
2. 画像が地味（テキストカードの単調さ）
3. 字幕がない（テロップだけ）
4. 全体的に動きがない（静止画+ken-burnsのみ）

## 制約

- **完全無料**：API課金不可（Edge TTS, Pillow, ffmpeg, Whisper, ローカルのみ）
- **GitHub Actions対応**：CI環境で動くこと
- **既存パイプラインとの整合性**：壊さない

## 改善項目（優先度順）

### 🔴 #1: 音声繋ぎの修正（音声優先パイプライン）

**問題：** シーンごとに別mp4をレンダリング → concatでギャップ・クリック音

**解決：**
1. 全シーンの音声を先に合成
2. ffmpegで音声を1本のmaster.wavに連結（ギャップなし、フェードあり）
3. EBU R128 loudnorm で音量正規化
4. 動画は「サイレントの映像トラック」として作る
5. 最後に master.wav を映像に重ねる（1ショット）

**期待効果：** クリック音消失、自然な流れ

---

### 🟠 #2: 字幕の自動生成と焼き込み

**問題：** テロップ（短いキーワード）だけで、ナレーション全文の字幕がない

**解決：**
1. 各シーンのナレーション文字列を ~14字単位で分割
2. シーンの実音声長に基づいて文字数比でタイムスタンプ配分
3. ASS（ASS subtitle format）で字幕を生成
   - 太字白文字、黒アウトライン、画面下中央、~52px
4. ffmpeg subtitles フィルタで焼き込み

**代替案：** faster-whisperで音声を再認識して word-level timestamps を取る（より正確、ただしCPU重い）

→ 第一段階は文字数比配分。重ければwhisper化を検討。

---

### 🟠 #3: シーン間クロスフェード

**問題：** ハードカットで切り替わるので安っぽい

**解決：** ffmpeg `xfade` フィルタ（0.4秒のディゾルブ）

```
[v0][v1]xfade=transition=fade:duration=0.4:offset=D0[vt1];
[vt1][v2]xfade=...
```

音声優先パイプラインでは映像のみクロスフェード、音声は連続していて違和感なし。

---

### 🟡 #4: BGM追加

**問題：** 音声のない時間帯（終盤フェードアウト等）で寂しい

**解決：**
1. CC0またはYouTube Audio LibraryからフリーBGMを assets/bgm/ に1本格納
2. ffmpegで-22dB（控えめ）でミックス
3. 動画開始0.5秒でフェードイン、最後1.5秒でフェードアウト

**ライセンス対策：** 著作権フリーのみ使用（Pixabay Musicは要確認、YouTube Audio LibraryのCC0楽曲が安全）

---

### 🟡 #5: 動きの追加（showwaves視覚化）

**問題：** 静止画+ken-burnsだけで「動き」が乏しい

**解決：** ffmpeg `showwaves` フィルタで音声波形を画面下部にオーバーレイ
- 高さ80px、幅1920px、画面最下部
- 音声に同期して動く → 視覚的な「生きている感」

加えて：
- チャプター番号のフェードイン演出
- 下部テロップのスライドイン

---

### 🟢 #6: テキストカードの高級化

**問題：** グラデーション背景+大きい文字、だけでは退屈

**解決：**
1. **微細なグリッドパターン**を背景にオーバーレイ（newspaper風）
2. **ストーリー別の差し色**（赤・青・緑・紫の4種ローテ）
3. **タイポグラフィ階層**（巨大ヘッドライン + 細いサブテキスト）
4. **左下に時刻スタンプ風の小さい数字**（番組らしさ）

---

### 🟢 #7: サムネイルの改善

**問題：** 文字大きいだけで「驚き」「ニュースっぽさ」が弱い

**解決：**
1. 「速報」「BREAKING」のラベルを赤背景で
2. 主要トピックの単語を黄色強調
3. 影と立体感を強化

---

## 新規依存ライブラリ

- `faster-whisper`（オプション、字幕の高精度化）

その他は既存（edge-tts、Pillow、ffmpeg）で完結。

## アーキテクチャの再設計

```
[Phase 1: Synthesize]
  各シーンのナレーション → audio_*.mp3
  ffprobeで実時間測定

[Phase 2: Audio Master]
  全audio_*.mp3 → master.wav（ギャップなし、loudnorm）
  シーンタイムライン算出 [(start_t, end_t), ...]

[Phase 3: Subtitles]
  ナレーション + タイムライン → subtitles.ass

[Phase 4: Visual Generation]
  各シーンの画像生成（Pillow）

[Phase 5: Visual Composition]
  画像 + ken-burns + xfade + showwaves + 字幕焼き込み + master.wav + BGM
  → output.mp4
  （1つのffmpegコマンドで構築）

[Phase 6: Validate]
  ffprobeで検証
```

## 検証ポイント（実装後）

- [ ] 音声にクリック音がないか
- [ ] 字幕が実音声と同期しているか
- [ ] クロスフェードが滑らかか
- [ ] BGMが邪魔しすぎないか
- [ ] showwavesが意味のある動きをしているか
- [ ] サムネが他チャンネルと並んだ時に見劣りしないか

## 残る無料の限界

- AI生成カスタム画像（DALL-E等は有料）
- AIアバター/VTuber（Runway等は有料）
- プロ品質の自然な日本語ナレーション（ElevenLabsは有料、Edge TTSは「機械的な落ち着いたナレーション」が限界）

これら3つを除けば、無料でかなりのクオリティに到達可能。
